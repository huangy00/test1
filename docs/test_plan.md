# OpenCart 电商系统测试计划

## 1. 项目概述

| 项目 | 说明 |
|------|------|
| 被测系统 | OpenCart 4.1.0.3 |
| 测试环境 | 本地 XAMPP (Apache + MySQL) |
| 测试范围 | 用户模块、商品模块、购物车模块、订单模块、支付模块 |
| 测试类型 | 接口自动化、UI自动化、数据库校验 |

## 2. 测试目标

- 验证核心业务流程（注册→登录→搜索→加购→下单→支付）的正确性
- 验证接口参数校验、异常处理、权限控制
- 验证数据库数据一致性（订单、库存、余额）
- 发现并记录缺陷，确保系统质量

## 3. 测试范围

### 3.1 用户模块

| 功能 | 测试点 | 优先级 |
|------|--------|--------|
| 注册 | 正常注册、邮箱重复、必填项为空、密码强度 | P0 |
| 登录 | 正常登录、错误密码、账号不存在、邮箱格式错误 | P0 |
| 个人信息 | 查看/修改个人信息、修改密码 | P1 |

### 3.2 商品模块

| 功能 | 测试点 | 优先级 |
|------|--------|--------|
| 商品列表 | 分页、排序、分类筛选 | P1 |
| 商品搜索 | 正常搜索、空结果、特殊字符、SQL注入 | P0 |
| 商品详情 | 价格显示、库存显示、图片加载 | P1 |

### 3.3 购物车模块

| 功能 | 测试点 | 优先级 |
|------|--------|--------|
| 加购 | 正常加购、库存不足、未登录加购 | P0 |
| 修改数量 | 增加/减少数量、超出库存 | P0 |
| 删除商品 | 正常删除、清空购物车 | P1 |

### 3.4 订单模块

| 功能 | 测试点 | 优先级 |
|------|--------|--------|
| 下单 | 正常下单、地址为空、库存不足 | P0 |
| 订单列表 | 查看历史订单、订单详情 | P1 |
| 订单取消 | 取消未支付订单 | P2 |

### 3.5 支付模块

| 功能 | 测试点 | 优先级 |
|------|--------|--------|
| 选择支付方式 | COD、Free Checkout | P0 |
| 支付确认 | 确认支付、重复提交 | P0 |

## 4. 测试策略

### 4.1 接口测试

- 覆盖所有 REST API 端点
- 每个接口覆盖：正常、异常参数、边界值、权限校验
- 使用 Pytest + Requests 框架
- 数据驱动：测试数据从 JSON 文件读取

### 4.2 UI 测试

- 覆盖核心业务流程（P0场景）
- 使用 Playwright + POM 设计模式
- 失败自动截图附加到 Allure 报告

### 4.3 数据库测试

- 订单创建后验证数据落库
- 支付完成后验证库存扣减、余额变更
- 多表关联查询验证数据一致性
